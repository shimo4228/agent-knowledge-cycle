# ADR-0003: Three-Layer Distillation (Raw → Knowledge → Identity/Rules)

> **Summary.** Agent memory is organized in three layers — raw episode log, distilled knowledge, identity and rules — each a pure function of the layer below plus a template. Layers 2 and 3 are disposable and can be regenerated from Layer 1.

## Status
accepted

## Date
2026-04-09

## Context

A naive agent memory stores everything at one level: a pile of facts, prompts,
and summaries that grows without structure. Retrieval then competes for the
same context window as fresh reasoning, and the operator has no way to ask
"what does the agent believe, vs. what has it merely observed?"

AKC needs a memory architecture that separates what happened, what patterns
were learned from it, and who the agent has become as a result — so that each
can be audited, pruned, and regenerated independently.

## Decision

Memory is organized in three layers, each a pure function of the layer below
plus a template:

```
┌─────────────────────────────────────────┐
│ Layer 3: Identity / Rules               │  Who the agent is.
│ (identity.md, rules/*.md)               │  Changes rarely. Human-reviewed.
└───────────────▲─────────────────────────┘
                │ distill-identity / rules-distill
┌───────────────┴─────────────────────────┐
│ Layer 2: Knowledge                      │  What patterns recur.
│ (knowledge.json)                        │  Changes with distillation.
└───────────────▲─────────────────────────┘
                │ distill
┌───────────────┴─────────────────────────┐
│ Layer 1: Episode Log                    │  What happened.
│ (logs/*.jsonl)                          │  Append-only. Source of truth.
└─────────────────────────────────────────┘
```

### Layer 1 — Episode Log (raw)

Defined by ADR-0002. The only layer that is written to at runtime during
normal agent operation. No distillation, no summarization, no reinterpretation.

### Layer 2 — Knowledge

A JSON store of distilled patterns, each carrying a `pattern` string, a
`distilled` timestamp, an `importance` score (0.0–1.0), and a `category` tag.
Generated by running distillation over the episode log. Patterns are
time-decayed on read (`importance * 0.95 ** days_elapsed`) so recent evidence
dominates stale signals without deletion.

See [`schemas/knowledge.schema.json`](../../schemas/knowledge.schema.json).

### Layer 3 — Identity and Rules

Human-readable markdown describing who the agent is and what principles guide
it. Generated by distilling Layer 2 through a second pass (see ADR-0004). All
changes pass through the Human Approval Gate (ADR-0005). Rules are the
deterministic layer — they load every session — while Layer 2 is probabilistic
context injection.

### Layer purity

Each layer depends only on the layer below plus a template, and never writes
back. This gives us:

- **Reproducibility** — re-running distillation on the same log yields the
  same (or equivalent) knowledge, which yields the same identity.
- **Replaceability** — swap templates to try a different persona, set of
  rules, or scoring function without touching the log.
- **Auditability** — "where did this belief come from?" is answered by tracing
  downward, not sideways.

## Alternatives Considered

- **Single-layer memory (all facts, equal weight)** — Simpler but conflates
  observation with belief. The operator cannot tell whether the agent has
  *learned* something or merely *seen* it once.
- **Two layers (log + summary)** — Works, but collapses "patterns" and
  "identity" into one document that is too big to audit and too small to
  separate concerns.
- **Graph memory (nodes + edges)** — Expressive, but expensive to build and
  hostile to human review. Deferred until the simpler design proves
  insufficient.

## Consequences

- Layer 2 and Layer 3 are **disposable**. Deleting them and re-running
  distillation from Layer 1 is a supported recovery path.
- The operator has three distinct pruning knobs: log retention, knowledge
  importance threshold, rule approval rate.
- Distillation cost is bounded by log size, not by cumulative knowledge size.
